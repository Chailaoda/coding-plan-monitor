# 安装与使用说明

多模型平台额度桌面监控悬浮窗。

## 一、首次安装

### 推荐方式：自动模式（无需手动导出 Cookie）

只要你**已经在 360 浏览器（UOS 版）登录过对应平台**，即可一键启动：

```bash
cd ~/Desktop/工作区/coding-plan-monitor
python3.11 test_api.py     # 验证火山方舟
python3.11 main.py          # 直接启动
```

程序会自动从 `~/.config/com.360.browser/Default/Cookies` 读取并解密 Cookie。各平台会自动匹配对应域名的 Cookie：

| 平台 | 关键 Cookie 字段 |
|------|-----------------|
| 火山方舟 | digest, csrfToken |
| MiMo | serviceToken |
| 阶跃星辰 | Oasis-Token, Oasis-Webid, INGRESSCOOKIE |

默认只启用火山方舟。启动后点击标题栏"标的"按钮可勾选其他平台。

如需自定义 UI 位置/阈值/通知开关，创建 `config.json`：

```bash
cp config.example.json config.json
chmod 600 config.json
```

### 备用方式：手动模式（不依赖 360 浏览器）

参考同目录下的 `cookie导出指南.md` 提取 Cookie，填入 config.json：

```json
{
  "auth": { "mode": "manual", "cookie": "...", "csrf_token": "..." }
}
```

### 验证

```bash
python3.11 test_api.py
```

应能看到三条带颜色的进度条。

## 二、手动启动

```bash
python3.11 ~/Desktop/工作区/coding-plan-monitor/main.py
```

桌面右上区域出现深色悬浮窗，每 60 秒自动刷新。

### 操作

| 操作 | 效果 |
|------|------|
| 拖动标题栏 | 移动悬浮窗（位置自动保存） |
| 点击左上角 ▼ / ▶ | 折叠/展开窗口 |
| 点击"标的" | 勾选监控平台 |
| 点击"更新" | 打开对应平台控制台 |
| 点击右上角 ✕ | 隐藏窗口（托盘图标保留） |

### 折叠态

折叠后标题栏显示：`火山 5小时:0.9% 周:3.6% | 阶跃星辰:2%`

### 各平台展开态显示

| 平台 | 展开态内容 |
|------|-----------|
| 火山方舟 | 5 小时 / 本周 / 本月 三行进度条 + 距重置时间 |
| MiMo | 百分比进度条 + 距到期时间 |
| 阶跃星辰 | 百分比进度条 + 剩余% + 有效期至 |
| 讯飞 MaaS | "待接入" + 打开控制台按钮 |

## 三、设置开机自启（systemd 用户服务）

```bash
mkdir -p ~/.config/systemd/user
cp ~/Desktop/工作区/coding-plan-monitor/coding-plan-monitor.service \
   ~/.config/systemd/user/
systemctl --user daemon-reload
systemctl --user enable --now coding-plan-monitor
```

查看状态/日志：

```bash
systemctl --user status coding-plan-monitor
journalctl --user -u coding-plan-monitor -f
```

卸载自启：

```bash
systemctl --user disable --now coding-plan-monitor
rm ~/.config/systemd/user/coding-plan-monitor.service
systemctl --user daemon-reload
```

**注意**：service 文件已写死 `DISPLAY=:0`，多数情况能正常工作。若看不到窗口，检查日志。

## 四、提醒规则

| 窗口 | 阈值 | 等级 | 行为 |
|------|------|------|------|
| 5 小时 | ≥70% | normal | 桌面通知 |
| 5 小时 | ≥90% | critical | 通知 + 铃声 + 窗口闪烁 |
| 本周 | ≥80% | normal | 桌面通知 |
| 本周 | ≥95% | critical | 通知 + 铃声 + 闪烁 |
| 本月 | ≥80% | normal | 桌面通知 |
| 本月 | ≥95% | critical | 通知 + 铃声 + 闪烁 |
| MiMo/阶跃星辰 | ≥60% | normal | 桌面通知 |
| MiMo/阶跃星辰 | ≥90% | critical | 通知 + 铃声 + 闪烁 |
| 认证连续 3 次失败 | — | normal | 通知"Cookie 可能已过期" |

每个周期内同一阈值只提醒一次（SQLite alert_history 去重）。

## 五、配置项说明

参考 `config.example.json`，主要配置项：

- `auth.mode`：`auto`（默认）或 `manual`
- `polling.interval_seconds`：拉取间隔（默认 60 秒）
- `targets.<id>.enabled`：启用哪些平台
- `alerts`：各窗口阈值、通知/铃声开关
- `ui`：窗口位置、透明度、置顶

## 六、数据存档

历史用量保存在 `usage.db`（SQLite），表结构：

- `usage_history`：每次拉取快照（ts, level, percent, reset_ts, …）
- `alert_history`：已触发提醒（防重复）

可用 `sqlite3 usage.db` 查看。

## 七、卸载

```bash
systemctl --user disable --now coding-plan-monitor 2>/dev/null
rm -f ~/.config/systemd/user/coding-plan-monitor.service
rm -rf ~/Desktop/工作区/coding-plan-monitor/
```

## 八、故障排查

### 悬浮窗不显示
- 命令行运行 `python3.11 main.py` 看错误输出
- 检查 `echo $DISPLAY`（应为 `:0` 或 `:1`）

### Cookie 读取失败
- 确认 360 浏览器已登录对应平台
- 阶跃星辰：先访问 `https://platform.stepfun.com` 再重启服务
- 手动模式参考 `cookie导出指南.md`

### 通知不弹
- 检查 `notify-send "测试" "正文"`

### 托盘图标不显示
- 需安装 `pystray` 和 `Pillow`：`pip3.11 install pystray Pillow`
- 不装托盘图标也不影响悬浮窗正常使用
